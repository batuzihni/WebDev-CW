General Structure and Layout Design:

HTML File:

Each of the HTML files (like profile.html, sitemap.html, content_ST4, etc.) is structured with a clean and semantic approach, using basic HTML tags and ensuring accessibility and usability. The header (<header>) contains a navigation bar with links to other pages. These links are defined within <nav> tags inside an unordered list (<ul>), which improves the semantic meaning of the code. The header is fixed at the top of the page to ensure it stays visible even when the user scrolls.

Profile Page:

Structure Overview:

The profile.html page displays personal or professional profile information, structured with a header, main content area, and a footer. The page uses semantic HTML elements to ensure the content is both structured and accessible.

Key HTML Tags:

  • <header>: Contains the website or page header, along with the name and title of the user/profile.
  • <article>: The primary content area of the page, which holds the main profile details, formatted with <h2> for subheadings, <p> for paragraphs, and <ul> for lists. This groups the profile data in a meaningful, semantic way.
  • <footer>: Contains contact information and external links to other social profiles, displayed in a visually separated section at the bottom of the page.

CSS Techniques:

  • Typography: Uses a clean, legible font family (like Arial or Verdana) for readability, with bold headings and well-spaced sections.
    • CSS Properties: font-family: Arial, sans-serif; margin: 20px 0; padding: 10px;
  • Image Styling: The profile picture is circular, achieved using the border-radius property.
    • CSS Properties: border-radius: 50%; This gives the profile picture a round appearance.
  • Grid System: A flexible grid layout is used for positioning profile details and images, ensuring proper alignment across screen sizes.
    • CSS Properties: display: grid; grid-template-columns: 200px 1fr; Ensures side-by-side alignment on larger screens and a stacked layout on smaller screens.

Key Features and Functionality:

  • Responsive Design: Media queries adjust the layout on smaller devices for a user-friendly experience.
  • Contact Information Section: External links and contact details are styled distinctly for clear visual cues.

Content Page:

Structure Overview:

The content_ST4.html page is part of a content-driven website and is structured to display detailed information in a clear and accessible manner. It uses HTML5 semantic tags and is designed to be responsive and user-friendly.

Key HTML Tags:

  • <header>: Contains the navigation bar and page title, fixed at the top for easy access.
  • <main>: Encloses the main content, including various sections that explain the core topic.
  • <section>: Divides the page into logical sections, each styled individually with CSS.
  • <footer>: Contains metadata and contact information at the bottom.

CSS Techniques:

  • Grid Layout: Uses a CSS Grid for a responsive two-column layout that adapts to screen size.
    • CSS Properties: display: grid; grid-template-columns: 1fr 1fr; Creates two equal-width columns on larger screens, switching to one column on smaller devices.
  • Card Design: Sections are styled as cards using box-shadow, padding, and border-radius to create a visually separated appearance.
    • CSS Properties: box-shadow: 0px 4px 8px rgba(0, 0, 0, 0.2); border-radius: 8px; Gives cards a 3D look with rounded corners.

Key Features and Functionality:

  • Responsive Design: Flexbox and media queries ensure a flexible layout across devices.
  • Navigation: A top navigation bar remains visible as users scroll.
  • Interactive Elements: Hover effects on cards and navigation items provide visual feedback.

Challenges And Lessons Learned:

Challenges:

  • Layout Responsiveness: Ensuring pages like content_ST4.html, profile.html, and sitemap.html rendered well across all devices was challenging due to fixed widths and layout issues on mobile screens.
  • Solution: Implemented meta viewport tags and used flexible layouts (flexbox and grid) to handle resizing smoothly.
  • Linking and Navigation Structure: Managing the navigation between pages was challenging as some links initially did not function as expected.
  • Solution: Ensured each link had the correct href attribute and used relative paths. In sitemap.html, SVG links with <a> tags were used to represent nodes.

Lessons Learned:

  • Use of Flexbox and CSS Grid: Learned the importance of flexible layout techniques for responsive design.
  • Simplification: Keeping HTML structure simple and using semantic tags improved code readability and accessibility.
